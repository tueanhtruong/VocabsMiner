# Implementation Plan: AI Vocabulary Extraction from Reading Passages

**Branch**: `001-vocab-extraction` | **Date**: 2026-06-12 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-vocab-extraction/spec.md`

## Summary

VocabsMiner lets IELTS learners paste an academic passage and instantly receive a ranked
list of Band 7+ vocabulary items ŌĆö each with word, part of speech, definition, and its
original context sentence. Extracted items can be saved to a personal "Vocabulary Goldmine"
collection backed by Firestore, gatekept by Firebase Auth.

**Technical approach**: A Next.js App Router application with a single server-side Route
Handler (`POST /api/extract`) that calls the OpenAI API (`gpt-4o-mini` with `json_object`
response format) to perform structured extraction. All Firestore reads/writes are
encapsulated in service modules under `lib/`. Auth state is shared via a React Context
provider rendered in the root layout. The home page (`app/page.tsx`) is the extraction
surface and works for both authenticated and unauthenticated visitors.

## Implementation Phases

> Tasks are dependency-ordered. Each task ends with `pnpm lint` (constitution Principle III).

### Phase 1 ŌĆö Foundation & Types (no external dependencies)

**Task 1.1 ŌĆö Type definitions** (`types/vocab.ts`)
- Define `VocabItem` interface: `word`, `partOfSpeech`, `definition`, `contextSentence`, `bandRelevance`
- Define `SavedVocabEntry` (extends `VocabItem`): adds `id`, `userId`, `savedAt`
- Define `ExtractionResponse`: `{ items: VocabItem[] }` and `ExtractionError`: `{ error: string }`
- Run lint

**Task 1.2 ŌĆö Firebase client singleton** (`lib/firebase/client.ts`)
- Initialize Firebase app with guard (`getApps().length === 0`) to prevent hot-reload re-init
- Export `auth` (getAuth) and `db` (getFirestore) singletons
- Reads `NEXT_PUBLIC_FIREBASE_*` env vars
- Run lint

**Task 1.3 ŌĆö Auth context** (`lib/auth/context.tsx`, `app/providers.tsx`)
- `AuthContext` with `user`, `loading`, `signIn`, `signOut`, `register` methods
- `onAuthStateChanged` listener in a `useEffect`
- `AuthProvider` wraps children; exported from `app/providers.tsx`
- Mount `<Providers>` in `app/layout.tsx`
- Run lint

**Task 1.4 ŌĆö Vocab Firestore service** (`lib/vocab/service.ts`)
- `saveVocabItem(uid, item: VocabItem): Promise<void>` ŌĆö writes to `users/{uid}/savedVocab/{word.toLowerCase()}`; duplicate-safe (Firestore `setDoc` is idempotent by document ID)
- `removeVocabItem(uid, word: string): Promise<void>` ŌĆö deletes the document
- `getVocabCollection(uid): Promise<SavedVocabEntry[]>` ŌĆö queries the subcollection, ordered by `savedAt` desc
- Run lint

### Phase 2 ŌĆö AI Extraction API (Story 1 ŌĆö P1)

**Task 2.1 ŌĆö Install OpenAI SDK**
- `pnpm add openai`
- Run lint

**Task 2.2 ŌĆö Extraction Route Handler** (`app/api/extract/route.ts`)
- `POST /api/extract` accepts `{ passage: string }`
- Validation: reject if `< 50 words` ŌåÆ 400 with message; reject if `> 2000 words` ŌåÆ 400 with limit message; reject non-English hint (basic heuristic or rely on AI)
- Call `gpt-4o-mini` with `response_format: { type: "json_object" }` and a structured system prompt that instructs extraction of 5ŌĆō20 Band 7+ items with all four required fields
- Parse and validate the JSON shape before responding
- Return `{ items: VocabItem[] }` on success; `{ error: string }` on failure
- Handle `OpenAI.APIError`, `SyntaxError`, and generic errors; never expose stack traces
- Run lint

**Task 2.3 ŌĆö PassageInput component** (`components/extraction/PassageInput.tsx`)
- `"use client"` textarea with live word count
- Shows "Too short" warning inline if `< 50 words`
- Shows "Too long" warning if `> 2000 words`
- "Extract" button disabled while loading or word count invalid
- On submit: calls `POST /api/extract`, sets loading state, exposes `onResults` callback
- Preserves pasted text on error (spec AC 1.5)
- Run lint

**Task 2.4 ŌĆö VocabResults component** (`components/extraction/VocabResults.tsx`)
- `"use client"` list rendering `VocabItem[]`
- Each card: word (bold), part of speech (badge), definition, context sentence (italicised blockquote), band relevance label
- Shows empty-state message when no Band 7+ items detected (spec AC 1.4)
- "Save" button per item ŌĆö disabled + "Saved Ō£ō" if word already in collection (requires `savedWords: Set<string>` prop)
- Unauthenticated save attempt ŌåÆ fires `onSaveAttemptUnauthenticated` prop
- Run lint

**Task 2.5 ŌĆö Home page wiring** (`app/page.tsx`)

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| AI model | `gpt-4o-mini` | ~$0.00015/req vs $0.003 for gpt-4o; more than sufficient for structured vocab extraction |
| JSON mode | `response_format: { type: "json_object" }` | Guarantees valid JSON; simpler than function-calling for a fixed output schema |
| Deduplication | Firestore doc ID = `word.toLowerCase()` | `setDoc` is idempotent; no extra query needed for FR-007 |
| Firebase init guard | `getApps().length === 0` | Prevents "duplicate app" error during Next.js hot reloads |
| Auth state sharing | React Context in `app/providers.tsx` | Only approach compatible with Client Components and RSC boundaries without a state library |
| Goldmine data fetch | Client-side `useEffect` in `VocabCollection` | Firebase client SDK cannot run in RSC; avoids need for Firebase Admin SDK (YAGNI) |
| Sign-in error message | Always "Invalid email or password" | FR-012: must not reveal whether email is registered |
